BUG 1 — SEGURIDAD (menu.routes.js)
------------------------------------
Problema:  POST /menu no tenía verifyToken
           Cualquiera podía agregar platos sin autenticarse
Corrección: router.post('/', verifyToken, verificarDatosPlato, menuController.agregarPlato)

BUG 2 — CONFIGURACIÓN (database/connection.js)
------------------------------------------------
Problema:  Importaba mongoUri de config pero luego usaba
           process.env.MONGO_URI directo — ignoraba el import
           dotenv no había cargado aún en ese punto
Corrección: usar mongoUri (el import) directamente en mongoose.connect()

BUG 3 — LÓGICA (services/plato.service.js)
--------------------------------------------
Problema:  findByIdAndUpdate sin { new: true }
           El cliente recibía los datos ANTES de la actualización
           aunque la actualización sí se guardaba en MongoDB
Corrección: Plato.findByIdAndUpdate(id, data, { new: true })
